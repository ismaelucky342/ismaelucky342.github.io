cpp module 01



cpp_module_01 delves deeper into

C++ programming, focusing on memory

allocation and management.


It introduces pointers to members,

references, and the use of switch

statements for control flow.


The goal is to enhance understanding

of dynamic memory and structured

programming techniques. Mastering

these concepts is essential for tackling

more advanced topics in future modules.


Additionally, this module emphasizes

the importance of constructors and

destructors in managing object lifecycles.


Students will also explore the nuances

of operator overloading, enabling more

intuitive and efficient code.


By the end of cpp_module_01, learners

will have a solid foundation in memory

management and object-oriented design.




Exercise 00: BraiiiiiiinnnzzzZ

This exercise involves creating a Zombie class in C++.

The class includes a private string attribute for the zombie's

name and a member function void announce() that makes the zombie

announce itself.

Additionally, two functions are implemented:


-> Zombie* newZombie(std::string name): Creates a zombie, names it,

and returns it for use outside the function scope.


-> void randomChump(std::string name): Creates a zombie, names it,

and makes it announce itself.


Objective: Understand when to allocate zombies on

the stack versus the heap and ensure proper destruction of zombies

with a destructor that prints a debugging message.

Exercise 01: Moar brainz!

Description: The "Moar brainz!" exercise involves creating a

horde of Zombies in C++.

The goal is to allocate multiple Zombie objects in a single

allocation and initialize them with a given name.


Objective:


-> Implement the function zombieHorde.

-> Allocate N Zombie objects and initialize them with the

provided name.

-> Ensure proper memory management by deallocating the zombies

using delete.

-> Test the function by calling announce() for

each zombie in the horde.

-> Check for memory leaks to ensure proper cleanup.

Exercise 02: HI THIS IS BRAIN

Description: Write a program that demonstrates the use of pointers

and references in C++.

The program must include:

-> A string variable initialized to "HI THIS IS BRAIN".

-> A pointer to the string (stringPTR).

-> A reference to the string (stringREF).

The program should print:

-> The memory address of the string variable.

-> The memory address held by stringPTR.

-> The memory address held by stringREF.

-> The value of the string variable.

-> The value pointed to by stringPTR.

-> The value pointed to by stringREF.

Objective: Understand the differences and similarities between

pointers and references in C++.

Exercise 03: Unnecessary violence

Description: Implement a Weapon class and two Human classes

(HumanA and HumanB) to demonstrate the use of references

and pointers.


The Weapon class should include:

-> A private attribute type (string).

-> A getType() member returning a constant reference to type.

-> A setType() member to update the type.


Human-A and Human-B should have:

-> A name and a Weapon.

-> An attack() that displays "< name > attacks with their weapon".


Objective: Demonstrate the differences between using references

and pointers in C++.

Exercise 04: Sed is for losers

Description: Create a program that replaces all occurrences

of a string (s1) with another string (s2) in a file.


The program should:

-> Take three parameters: a filename, s1, and s2.

-> Create a new file named .replace with the modified content.

-> Handle unexpected inputs and errors gracefully.


Objective: Practice file manipulation and string

operations in C++.

Exercise 05: Harl 2.0

Description: Automate Harl's complaints using a class with member

functions for different log levels.


The Harl class should include:

-> Private member functions: debug(), info(), warning(), and error().

-> A public complain(std::string level) function to call the appropriate private function based on the level.


Objective: Use pointers to member functions to avoid a forest of

if/else statements.

Exercise 06: Harl filter

Description: Implement a system to filter Harl's complaints based

on log levels.


The program should:

-> Take one parameter: a log level.

-> Display all messages from the specified level and above.

-> Handle invalid log levels gracefully.


Objective: Use the switch statement to filter log levels

effectively.
