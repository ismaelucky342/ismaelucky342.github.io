cpp module 02



cpp_module_02 dives into more advanced object-oriented programming in C++.

It introduces the Orthodox Canonical Form and the concept of fixed-point numbers.


Students will gain experience with operator overloading and ad-hoc polymorphism.

They'll also build reusable and correctly structured C++ classes split into .hpp and .cpp files.


This module emphasizes clean class design, memory management, and precise object behavior.

It culminates with a geometric challenge using a custom Point class and fixed-point arithmetic.


By completing cpp_module_02, learners strengthen their understanding of class mechanics,

encapsulation, and operator customization in C++.




Exercise 00: My First Class in Orthodox Canonical Form

Description: Create a class named Fixed following the Orthodox Canonical Form.

This includes: default constructor, copy constructor, copy assignment operator, and destructor.


Objective: Learn how to structure a well-formed C++ class with basic member functions.

Use internal representation with a fixed-point format and implement getRawBits() and setRawBits().

Exercise 01: Towards a more useful fixed-point number class

Description: Enhance the Fixed class to support conversions from and to int and float.

Add constructors that accept int and float, and member functions toFloat() and toInt().


Objective: Make the class practical and printable by overloading the << operator.

Exercise 02: Now we’re talking

Description: Overload a wide range of operators for the Fixed class:

-> Arithmetic: +, -, *, /

-> Comparison: >, <, >=, <=, ==, !=

-> Increment/Decrement: ++, -- (pre and post)


Objective: Understand ad-hoc polymorphism and write intuitive arithmetic logic for user-defined types.

Also implement static min() and max() functions for comparisons.

Exercise 03: BSP (Bonus)

Description: Create a Point class using Fixed values and implement the bsp function.

This function determines if a given point lies inside a triangle formed by three other points.


Objective: Apply your Fixed class in a geometric context using Binary Space Partitioning logic.

This is optional, but highly recommended to solidify your understanding of class usage in real applications.
