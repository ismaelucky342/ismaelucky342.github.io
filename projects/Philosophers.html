Philosophers



Philosophers is is a classic synchronization

problem in computer science.


We have a certain number of philosophers who are

sitting at a table and need to eat and think. Each

philosopher needs two forks to eat, but there

are only as many forks as there are philosophers.

The challenge is to design a protocol that ensures

no philosopher starves and that the system avoids

deadlocks.


The goal is to simulate the Dining Philosophers problem

ensuring that each philosopher can eat and think without

causing deadlocks and starvation.

The bonus extends the simulation by using processes and

semaphores, introducing a more complex synchronization

mechanism.

Philosophers - Mandatory part

Each philosopher is represented as a thread.

There is one fork between each philosopher, and each fork is

protected by a mutex.Philosophers must alternate between eating,

thinking, and sleeping.

The program must prevent deadlocks and ensure no philosopher

starves. The simulation ends when all philosophers have

eaten a specified number of times (if provided).


The implementation must adhere to the following constraints:

Philosophers - Bonus part

In the bonus part, philosophers are represented as processes

instead of threads.

The forks are placed in the center of the table and are

controlled by semaphores.

Each philosopher is a process, and the main process should

not be a philosopher.

Each philosopher must alternate between eating, thinking,

and sleeping.

The program must prevent deadlocks and ensure no philosopher

starves.

The simulation ends when all philosophers have eaten a

specified number of times (if provided).
